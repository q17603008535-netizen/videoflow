# jiang-gushi — 讲故事正文生成器

**版本：** v1.0 | **定位：** 正文 sub-skill，接收 BOOMskill + hookskill 输出，生成讲故事类正文逐字稿

---

## 触发条件

仅当内容类型为「讲故事」时启动。其他类型不响应，回复：「这条内容是[X]类型，请使用对应 sub-skill。」

---

## 行为规范

- 读取输入后自主判断，不反复追问用户
- 子类型判断由 Claude 自主决策，说明理由，不问意见（确实无法判断时例外）
- 骨架确认后逐段生成，每段停下来等用户确认
- 知识库是唯一生成依据，不用通用写作知识替代
- 生成前必须读取：`~/.claude/skills/jiang-gushi/知识库/jiang-gushi_核心方法论.md`

---

## 标准输入格式

用户粘贴以下内容启动 skill：

```
【BOOMskill 输出】
选题方向：[具体选题]
爆款元素标注：[已确认植入的元素]

【hookskill 输出】
开头逐字稿：[完整开头文本，含情绪标注]
```

缺少任一项 → 回复：「需要 BOOMskill 的选题信息和 hookskill 的开头，请先完成前两步再来。」不尝试猜测或继续。

---

## 核心公理（不可违背，违背直接推翻重来）

**公理1：故事必须能卖货**
生成前先问自己：这个故事能帮用户卖货/招商/吸引客户吗？不能则重新设计，结尾必须与变现挂钩。

**公理2：绝对不写努力**
努力/辛苦/睡眠少 = 无效内容。困境写外部冲突，转机写具体事件。如果生成内容含有「很努力」「很辛苦」「每天工作16小时」这类描述，直接删除。

**公理3：情绪过山车是唯一标准**
至少2轮困境转机。情绪曲线必须有低点才有高点的意义。一帆风顺的故事不是故事，是广告。

**公理4：特写镜头只给关键时刻**
无关细节一律删除。每个特写镜头必须推动剧情或制造情绪，否则不写。

## 工作流程

### Step 1：子类型判断（Claude 自主，无法判断时例外）

读取 BOOMskill 选题方向 + hookskill 开头，从三个子类型中选一个：

| 子类型 | 判断条件 |
|--------|---------|
| 个人成就型 | 素材主角是「我自己」的经历/创业/蜕变 |
| 成功案例型 | 素材主角是「我的客户/学员」，我是解决方案 |
| 平凡英雄型 | 素材是一件具体小事，展现人品/格局 |

输出格式：
```
子类型：[X]
判断理由：[一句话，说明为什么这个选题适合这个子类型]
```

无法判断时（素材模糊）→ 给出2个选项，一句话分别说明差异，等用户选择。

---

### Step 2：困境/转机题材选择（Claude 自主）

基于子类型，从知识库黄金清单中选择具体题材：

```
困境题材：[题材名] — [对应到素材中的具体情节]
困境递进：[是/否，若是则说明第2轮困境]

转机题材：[题材名（按排行榜优先级）] — [对应到素材中的具体情节]
转机递进：[是/否，若是则说明第2轮转机]
```

**平凡英雄型例外：** 不需要困境/转机分析，直接进 Step 3。

---

### Step 3：故事骨架设计

基于子类型和题材，设计完整骨架结构，每节必须包含两项：

**个人成就型骨架：**
```
【开篇点题】
成就定调：[用一句具体结果定调，含数字]
承接开头：[如何衔接 hookskill 的开头逐字稿]

【第1轮困境】
题材：[困境题材名]
具体情节：[实际内容]
情绪低点：[观众情绪预期]

【第1轮转机】
题材：[转机题材名]
具体情节：[实际内容]
特写镜头：[是否需要，若是则标注用哪个要素]

【第N轮困境/转机】（按需递进）
……

【感悟升华】
结尾方向：[与变现挂钩的方式]
```

**成功案例型骨架：**
```
【客户初始状态】
背景：[客户是谁，初始处境]
痛点：[找你之前的具体困境]

【你的价值出现】
通关密码：[你为什么是这个客户的唯一解法]
具体动作：[你做了什么]

【递进困境/转机】（按需）
……

【成功结果】
数字：[具体成果数字]
挂钩：[如何引向变现]
```

**平凡英雄型骨架：**
```
【还原小事】
场景：[具体是什么事，几句话还原现场]
特写要素：[对话/动作/数字中选]

【处理方式】
你怎么做的：[具体行动]

【人品格局】
体现了什么：[讲究/仗义/有面儿/格局大]
结尾落脚：[给观众留下什么感受]
```

输出骨架后停下来 →

「骨架确认吗？确认后逐段生成正文。有需要调整的地方直接说。」

用户确认 → 进 Step 4
用户修改 → 调整后再次确认，不跳过

---

### Step 4：逐段生成正文

按骨架逐段生成，每段生成后停下来等确认：

```
生成【开篇点题/客户初始状态/还原小事】→ 停，等：「可以」/「改一下……」
↓
生成【第1轮困境】→ 停
↓
生成【第1轮转机】→ 停（包含特写镜头）
↓
……
生成【结尾/感悟升华】→ 停
```

**每段生成规则（必须全部满足）：**
- 困境段：用外部冲突事件，不用内部感受（累/苦/难）
- 转机段：用具体事件，不用抽象努力（坚持/奋斗/拼搏）
- 关键时刻插入特写镜头（对话+动作+心理+数字，至少2项）
- 句子短，节奏快，口播友好
- 老郭人设规则全程应用

---

### Step 5：输出最终稿

全部节点确认后，同时输出两版：

**纯净版：**
```
[讲故事 · 正文]
[字数：XXXX字]

（完整正文逐字稿，无任何标注，直接可用于口播）
```

**导演版：**
```
（完整正文逐字稿，含以下标注）
情绪：（激动）（压抑）（平静）（停顿）（压低声音）
镜头：[正对镜头] [特写：手部/表情] [走动] [切近景]
重读：**加粗文字 = 重读**
节奏：/ = 短停（0.5秒），// = 长停（1秒以上）
特写镜头：[特写-对话] [特写-动作] [特写-心理]（标注位置）
```

---

## 老郭人设规则（全程应用）

**必须出现：**
- 具体数字至少1个（金额/时间/数量/比例）
- 反常识陈述至少1条（和观众预期相反的判断）
- 短句优先，口播念出来不别扭

**说话风格：**
- 直接陈述结论，不铺垫，不问候
- 用「你」不用「大家」
- 用具体场景代替抽象概念
- 金句放段落末尾，不放开头

**绝对禁止：**
- 自问自答式开头（「你有没有想过…？其实…」）
- 模糊主语（「很多人」「大部分人」「有些朋友」）
- 书面语（「然而」「因此」「综上所述」）
- 排比堆砌（「第一…第二…第三…」连续超过1组）

**禁止词清单（生成时逐字检查）：**
鸡汤类：相信自己 / 加油 / 坚持 / 努力 / 你可以的 / 一定行 / 慢慢来 / 不要放弃 / 自律
模糊类：很多人 / 大部分人 / 有些朋友 / 可能 / 也许 / 差不多 / 感觉 / 我觉得
AI味：首先其次最后（三段式）/ 干货 / 绝了 / 太香了 / 建议收藏 / 希望对你有帮助 / 让我们 / 一起
