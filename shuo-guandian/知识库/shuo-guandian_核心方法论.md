## 一、底层逻辑：为什么说观点最能变现

- 用户不为知识付费，只为**喜欢的人**消费
- 专业≠喜欢，有用≠会买，知识≠魅力
- 有观点才有真粉丝，真粉丝才能变现
- 100万强IP博主（80-90%真爱率）远超3000万知识博主（不到4%真爱率）

## 二、核心创作思路（唯一黄金法则）

> **给他想做的事找个理由，给他不想做或没做成的事找个借口。**

三个字总结：**支持他**

- 不讲客观道理，不当和事佬
- 像对待朋友/恋人一样**无条件支持**粉丝
- 你的观点=替那群人说出了他们说不出来的话
- 你成为情绪领袖，才能收钱（参考：寺庙/教堂模型）

### 支持的三种实战场景

| 场景 | 粉丝困境 | 正确做法 |
|------|----------|----------|
| 想出去工作被阻拦 | 想做没做成 | "凭什么你要受罪？你必须出去，找回自己！" |
| 装修完被嫌弃 | 做了被批评 | "事后诸葛亮，让他们自己装！" |
| 不想继续当公务员 | 想做但犹豫 | "30岁每天盖戳，干到60岁？公务员是给没梦想的人的" |

## 三、选题方法

### 第一步：确定写作对象（目标粉丝群体）

常见写作对象：年轻女性 / 全职妈妈 / 结婚的女人 / 年轻男性 / 买车的人

**原则：谁说了算，就向着谁说话**（教育机构老师判断谁掌钱的类比）

### 第二步：从5个维度找选题

| 维度 | 说明 | 示例 |
|------|------|------|
| 过错 | 帮他赎罪 | "恋爱脑没有错，是世界错了" |
| 纠结 | 帮他解决纠结 | "30岁没结婚，丢人吗？" |
| 委屈 | 帮他发声 | "全职妈妈有什么出息？" |
| 麻烦 | 帮他解决麻烦 | "带朋友买车，没个能买的" |
| 伤害 | 替他反击 | "闺蜜老贬低我怎么办？" |

## 四、脚本结构（两种）

### 结构1：归纳法（主流）

```
开篇表明观点 → 论据1 → 论据2 → 论据3 → 强化观点
```

- 论据数量：2-5个（3个最常见）
- 每个论据展开约20-30秒口语量
- 论据类型：个人经历小故事 / 具体事件案例 / 常见现象描述 / 对比论证

**示例骨架（男人都很专一）：**
- 开篇：世界对男人最大的误解是不专一
- 论据1：对美女专一（18岁喜欢，80岁还喜欢）
- 论据2：对游戏专一（海贼王看20多年，刀塔打十几年）
- 论据3：对发型专一（永远只有一个发型）
- 强化：所以男人不专一是大误解

### 结构2：反证法

```
开篇表明观点 → 反例（不这样的坏处）→ 正例（这样的好处）→ 强化观点
```

- 反例要具体、有画面感
- 正例要有对比、显得正确
- 适合有争议的话题

**示例骨架（女生该不该物质）：**
- 开篇：该物质，太该了
- 反例：随便说"吃什么都行"→ 被带去吃沙县，还被羞辱
- 正例：把自己养得很贵→ 没人敢带去吃沙县
- 补充论据：自然择偶本能 / 物质=不低于自己原有标准=自尊
- 强化+反击：物质是人性，说女生物质就是反向承认自己能力不行

## 五、大流量原则：解气才涨粉

**核心：不是说服反方，而是支持正方**

- 哪一方人多，就站哪一方
- 解气=涨粉，憋屈=掉粉
- 打回去 > 告老师，反抗 > 顺从

**判断方式：**
- "告老师"→ 小流量（少数人认同）
- "打回去"→ 大流量（大多数人认同）
- "老板打人不对"→ 大流量（不管3000万损失）
- "老板能理解"→ 被骂

## 六、论据来源三种方法

### 方法1：对着手机说（最推荐）
1. 确定观点
2. 对手机说"我觉得XXX，为什么？我说5个点"
3. 硬逼自己说完，录音转文字
4. 至少有2个能用，选最好的2-3个展开

### 方法2：从评论区找（最省时）
1. 搜索相同话题热门视频
2. 找点赞多且和你态度相同的评论（选2-4条）
3. 每条评论补充1-2句话，加个人化表达
4. 2-4条评论 = 2-4个论据

### 方法3：查资料补充
- 写小经历：「我有个朋友/姐妹，她也经历了…」
- 写小事件：「有一次我看到/听说一个事…」
- 写我会怎么做：「如果是我，我会…」
- 民法典/法律规定（百度搜索）
- 易懂官方数据：「根据调查显示，XX%的人…」

## 八、老郭人设硬性规则

**必须有：**
- 具体数字（≥1个）
- 反常识陈述（≥1条）
- 短句节奏

**绝对禁止：**
- 自问自答式开头
- 模糊主语（很多人/大部分人）
- 书面语（然而/因此）
- 排比堆砌

**禁止词：**
相信自己 / 加油 / 坚持 / 努力 / 你可以的 / 很多人 / 大部分人 / 可能 / 也许 / 干货 / 绝了 / 首先其次最后 / 建议收藏
