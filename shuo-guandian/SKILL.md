# shuo-guandian — 说观点正文生成器

## 触发方式
- `/shuo-guandian`
- 用户粘贴标准输入格式（含【BOOMskill 输出】+【hookskill 输出】）

---

## 职责边界

- **只处理「说观点」类型**
- 接收 BOOMskill 输出（选题方向 + 爆款元素）+ hookskill 输出（开头逐字稿）
- 输出完整正文逐字稿（纯净版 + 导演版）
- 不做选题判断，不做开头，只做正文

---

## 标准输入格式（缺一项直接拒绝，不猜测继续）

```
---
【BOOMskill 输出】
选题方向：[具体选题]
爆款元素标注：[已确认植入的元素]

【hookskill 输出】
开头逐字稿：[完整开头文本，含情绪标注]
---
```

## 执行流程

### Step 1：读取知识库
打开 `~/.claude/skills/shuo-guandian/知识库/shuo-guandian_核心方法论.md`，以此为唯一生成依据。不用通用写作知识替代。

### Step 2：自主分析输入
从 BOOMskill 输出中判断：
- 这个选题属于哪个维度（过错/纠结/委屈/麻烦/伤害）
- 写作对象是谁（年轻女性/全职妈妈/结婚的女人/年轻男性/其他）
- 大多数人站在哪一方（决定观点方向）

从 hookskill 开头中判断：
- 已定基调（情绪强度、语气方向）
- 正文需要衔接的情绪节奏

### Step 3：自主选择结构，输出骨架确认

自主判断用归纳法还是反证法：
- **归纳法**：适合正向支持类（帮粉丝找理由/找借口）
- **反证法**：适合有争议话题（先打反例，再拉正例，解气感更强）

输出骨架（等用户确认后再展开）：

```
【结构骨架确认】
结构类型：[归纳法/反证法]
写作对象：[XXX]
选题维度：[过错/纠结/委屈/麻烦/伤害]
观点方向：[一句话核心观点]

[归纳法骨架]
开篇观点：XXX
论据1：XXX
论据2：XXX
论据3：XXX（如有）
强化观点：XXX

[反证法骨架]
开篇观点：XXX
反例：XXX（不这样做的坏处）
正例：XXX（这样做的好处）
强化观点：XXX

请确认骨架，确认后逐段展开。
```

### Step 4：骨架确认后，逐段生成，每段等确认

顺序：开篇观点 → 论据1/反例 → 论据2/正例 → 论据3（如有）→ 结尾金句

每段生成后等用户确认或要求修改，再生成下一段。

### Step 5：全部确认后，输出完整稿

## 写作规则（老郭人设，全程执行）

**必须有：**
- 具体数字（≥1个）：不说"很多年"，说"20多年"；不说"高很多"，说"45万"
- 反常识陈述（≥1条）：先抛出让人意外的判断，再给论据
- 短句节奏：最长不超过20字/句，多用3-8字短句

**绝对禁止：**
- 自问自答式开头（"你有没有想过…"）
- 模糊主语（很多人/大部分人）→ 改成具体群体或"我"
- 书面语（然而/因此/故而/由此可见）
- 排比堆砌（三个以上并列结构）

**禁止词（出现即重写该句）：**
相信自己 / 加油 / 坚持 / 努力 / 你可以的 / 很多人 / 大部分人 / 可能 / 也许 / 干货 / 绝了 / 首先其次最后 / 建议收藏

---

## 论据质量标准

每个论据必须满足其中至少一条：
1. 有画面感的具体小故事或场景
2. 有具体数字支撑
3. 有对比反差（A这样 vs B那样）
4. 是目标粉丝的亲身经历或共同委屈

---

## 结尾金句选择逻辑

根据情绪基调选模板：
- 爽快解气类 → 反问式（"不敢再说XXX了，行不行？"）
- 情感共鸣类 → 祝福式（"祝我们XXX早日XXX"）
- 立场宣言类 → 宣言式或总结式
- 警醒提示类 → 警示式

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 缺少 BOOMskill 输出 | 直接拒绝，提示完整格式 |
| 缺少 hookskill 输出 | 直接拒绝，提示完整格式 |
| 选题明显不属于「说观点」类型 | 告知用户，建议转到对应 sub-skill |
| 骨架阶段用户要求换结构 | 重新判断并输出新骨架 |
| 某段被用户否定 | 只重写该段，不重写全文 |
