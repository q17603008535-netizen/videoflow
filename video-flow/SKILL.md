# video-flow — 短视频内容生产流水线总入口

**版本：** v1.0 | **定位：** 检测用户所处阶段，告知用户，自动执行对应流程

触发方式：`/video-flow`、直接粘贴任何阶段的输出内容、描述当前需求

---

## 行为规范

- 读取用户输入后自主判断阶段，不反复追问
- 判断完成后先告知用户当前阶段，再执行流程
- 无法判断时给出选项，一次选定，不来回确认
- 执行对应流程前必须读取该流程的 SKILL.md
- 不跳过任何阶段，不提前生成用户尚未到达的环节

---

## 阶段检测逻辑（按优先级顺序判断）

**第一优先：看用户粘贴了什么**

| 用户输入包含 | 判断阶段 | 执行流程 |
|---|---|---|
| 【BOOMskill 输出】+【hookskill 输出】两个块 | 正文阶段 | zhengwen-router |
| 【BOOMskill 输出】但无【hookskill 输出】 | 开头阶段 | hook-anchor |
| 录音转文字稿（大段口语化文字，无结构标记） | 建档阶段 | ip-archive |

**第二优先：看用户说了什么**

| 用户描述包含 | 判断阶段 | 执行流程 |
|---|---|---|
| 「新号」/「起号」/「建档案」/「没有人设」/「第一次」 | 建档阶段 | ip-archive |
| 「已有档案」/「选题」/「这个主题怎么做」/「帮我优化选题」 | 选题阶段 | boom-elements |
| 「选题定了」/「帮我写开头」/「怎么开场」 | 开头阶段 | hook-anchor |
| 「开头有了」/「帮我写正文」/「出逐字稿」 | 正文阶段 | zhengwen-router |

**无法判断时：**

一次性给出选项，等用户选定后执行：

```
你现在在哪个阶段？

1. 建档案 — 新号起步，有录音或想建人设
2. 选题 — 有方向，想让选题更有流量
3. 开头 — 选题定了，需要开场钩子
4. 正文 — 开头有了，需要完整逐字稿

选一个数字，我立刻开始。
```

---

## 告知格式（检测完成后必须先输出）

```
📍 当前阶段：[阶段名]
➡️ 进入流程：[对应 skill 名]

[一句话说明接下来要做什么]
```

输出告知后，读取对应 SKILL.md，立即开始执行，不等用户再次确认。

---

## 对应流程读取路径

| 阶段 | 读取文件 |
|---|---|
| 建档阶段 | `~/.claude/skills/ip-archive/SKILL.md` |
| 选题阶段 | `~/.claude/skills/boom-elements/SKILL.md` |
| 开头阶段 | `~/.claude/skills/hook-anchor/SKILL.md` |
| 正文阶段 | `~/.claude/skills/zhengwen-router/SKILL.md` |

读取后完全按照该 SKILL.md 的流程执行，不做简化，不跳步骤。

---

## 跨阶段处理

**用户想跳过某个阶段：**
直接接受，从用户指定的阶段开始，不强制回到上游。

**用户想重新从某个阶段开始：**
```
「从[X]阶段重新开始，是吗？」
```
确认后读取对应 SKILL.md，从头执行。

**用户在流程中途想切换阶段：**
识别切换意图 → 输出新的告知格式 → 读取新阶段 SKILL.md → 执行。
