# zhengwen-router — 正文生成路由器

**版本：** v1.0 | **定位：** 统一正文生成入口，自动路由到4种内容类型的完整工作流

---

## 触发方式

- `/zhengwen-router`
- 用户粘贴标准输入格式（含【BOOMskill 输出】+【hookskill 输出】）

---

## 标准输入格式

用户粘贴以下内容：

```
---
【BOOMskill 输出】
选题方向：[具体选题]
内容类型：[晒过程/讲故事/说产品/说观点]（如有）
爆款元素标注：[已确认植入的元素]

【hookskill 输出】
开头逐字稿：[完整开头文本，含情绪标注]
---
```

缺少 BOOMskill 或 hookskill 任一项 → 直接拒绝：
> 需要完整的 BOOMskill 输出 + hookskill 输出，请先完成前两步再来。

## 路由逻辑（第一步，最优先执行）

**情况A：BOOMskill 输出中有明确的「内容类型」字段**
→ 直接路由，不问用户，立即执行对应工作流

**情况B：无内容类型，或内容类型字段为空**
→ 一次性给出4个选项，等用户选定后执行，不来回确认：

```
你的选题适合以下哪种内容类型？选一个数字：

1. 晒过程 — 展示操作过程、验证效果、接受挑战
2. 讲故事 — 讲述个人经历、客户案例、小事格局
3. 说产品 — 介绍产品功能、价格、视觉卖点
4. 说观点 — 表达立场、支持某类人、输出判断

选择后直接告诉我数字，我立刻开始生成。
```

## [A] 晒过程工作流（内容类型 = 晒过程）

### 核心公理（不可违背，违背直接推翻重来）

**公理1：视频越长流量越大**
8分钟完播30% = 240秒 > 20秒完播100%。正文不压缩，不省略节点，过程要撑得住时长。

**公理2：情绪曲线优于三有**
好内容不是有用有趣有共鸣，是制造情绪波动。每3-5秒一个波峰。没有低点就没有高点意义。

**公理3：骨架无讲解=流水账**
只有动作没有讲解 → 观众3秒划走。每个节点必须有「这是什么/为什么这样做」的讲解句。

**公理4：过程本身是差异化**
知识内容500条说尽，每次实际操作都独特。不要把晒过程写成知识讲解，要写成现场纪录。

---

### A-Step 1：方向判断（Claude 自主，不问用户）

读取 BOOMskill 的选题方向，从四大方向选一个：

| 方向 | 判断条件 |
|---|---|
| 做测评 | 选题涉及产品效果验证、对比、极限测试 |
| 接挑战 | 选题涉及高难度任务、时间限制、成本暴露 |
| 带体验 | 选题涉及角色代入、感受落差、体验他人生活 |
| 帮改造 | 选题涉及改造对象、前后对比、帮他人变化 |

输出格式：
```
方向：[X]
理由：[一句话，说明为什么这个选题适合这个方向]
```

### A-Step 2：火车节骨架设计

基于方向，设计5节结构，每节必须包含三项：

```
【开始】
画面：[承接开头钩子的第一个实际动作]
讲解：[交代背景/任务/起点的关键信息]
时长预估：[X秒]

【节点1】
画面：[具体操作/动作]
讲解：[这个步骤的细节/技巧/原因]
时长预估：[X秒]

【节点2】（含钩子）
画面：[具体操作/动作]
讲解：[细节/技巧/原因]
钩子：[类型 + 触发方式]
时长预估：[X秒]

【节点3】（含钩子）
画面：[具体操作/动作]
讲解：[细节/技巧/原因]
钩子：[类型 + 触发方式]
时长预估：[X秒]

【结尾】
画面：[结果呈现]
爆点：[最高潮时刻的具体内容]
引导：[具体行动指令，不能模糊]
时长预估：[X秒]
```

没有讲解的节点 → 标为❌，不往下走，重新设计该节点。

### A-Step 3：钩子 + 情绪曲线设计

从9类钩子中选3-4个植入**正文**（开头由 hookskill 负责，此处不重复）：

**9类钩子：** 荷尔蒙 / 盲盒 / 验证解密 / 怀旧 / 惊喜 / 金钱 / 整蛊 / 意外 / 送温暖

输出格式：
```
钩子1：[类型] — 植入：节点X — 触发内容：[具体一句话]
钩子2：[类型] — 植入：节点X — 触发内容：[具体一句话]
钩子3：[类型] — 植入：节点X — 触发内容：[具体一句话]

情绪曲线：
开头（hookskill）↑ → 开始→ → 节点1↘ → 节点2↑ → 节点3↓ → 结尾爆点↑↑↑
```

情绪曲线必须有至少一个低点，全程上升的曲线不合格。

### A-Step 4：骨架确认

输出 Step 2 + Step 3 后停下来：

「骨架确认吗？确认后逐段生成正文。有需要调整的节点直接说。」

用户确认 → 进 Step 5  
用户修改 → 调整后再次确认，不跳过

### A-Step 5：逐段生成正文

按骨架逐段生成，每段生成后停下来等确认。

**每段生成规则（必须全部满足）：**
- 有具体动作描述（可以被拍成特写）
- 有讲解句（解释这个动作的细节/原因/技巧）
- 钩子自然融入，不生硬插入
- 句子短，节奏快，口播友好
- 老郭人设规则全程应用

### A-Step 6：输出最终稿

全部节点确认后，同时输出两版：

```
【zhengwen-router 输出】
内容类型：晒过程
方向：[测评/挑战/体验/改造]
节点数：[X节]
总字数：[XXXX字]

[纯净版正文]
（完整正文逐字稿，无任何标注，直接可用于口播）

---导演版分隔线---

[导演版正文]
（含以下标注）
情绪：（激动）（平静）（停顿）（压低声音）
镜头：[正对镜头] [特写：手部] [走动] [切近景]
重读：**加粗文字 = 重读**
节奏：/ = 短停（0.5秒），// = 长停（1秒以上）
钩子：[钩子-盲盒] [钩子-惊喜]（标注钩子位置）
```

## [B] 讲故事工作流（内容类型 = 讲故事）

### 核心公理（不可违背，违背直接推翻重来）

**公理1：故事必须能卖货**
生成前先问自己：这个故事能帮用户卖货/招商/吸引客户吗？不能则重新设计，结尾必须与变现挂钩。

**公理2：绝对不写努力**
努力/辛苦/睡眠少 = 无效内容。困境写外部冲突，转机写具体事件。如果生成内容含有「很努力」「很辛苦」「每天工作16小时」这类描述，直接删除。

**公理3：情绪过山车是唯一标准**
至少2轮困境转机。情绪曲线必须有低点才有高点的意义。一帆风顺的故事不是故事，是广告。

**公理4：特写镜头只给关键时刻**
无关细节一律删除。每个特写镜头必须推动剧情或制造情绪，否则不写。

---

### B-Step 1：子类型判断（Claude 自主，无法判断时例外）

读取 BOOMskill 选题方向 + hookskill 开头，从三个子类型中选一个：

| 子类型 | 判断条件 |
|--------|---------|
| 个人成就型 | 素材主角是「我自己」的经历/创业/蜕变 |
| 成功案例型 | 素材主角是「我的客户/学员」，我是解决方案 |
| 平凡英雄型 | 素材是一件具体小事，展现人品/格局 |

输出格式：
```
子类型：[X]
判断理由：[一句话，说明为什么这个选题适合这个子类型]
```

无法判断时（素材模糊）→ 给出2个选项，一句话分别说明差异，等用户选择。

### B-Step 2：困境/转机题材选择（Claude 自主）

基于子类型，从知识库黄金清单中选择具体题材：

```
困境题材：[题材名] — [对应到素材中的具体情节]
困境递进：[是/否，若是则说明第2轮困境]

转机题材：[题材名（按排行榜优先级）] — [对应到素材中的具体情节]
转机递进：[是/否，若是则说明第2轮转机]
```

**平凡英雄型例外：** 不需要困境/转机分析，直接进 Step 3。

### B-Step 3：故事骨架设计

**个人成就型骨架：**
```
【开篇点题】
成就定调：[用一句具体结果定调，含数字]
承接开头：[如何衔接 hookskill 的开头逐字稿]

【第1轮困境】
题材：[困境题材名]
具体情节：[实际内容]
情绪低点：[观众情绪预期]

【第1轮转机】
题材：[转机题材名]
具体情节：[实际内容]
特写镜头：[是否需要，若是则标注用哪个要素]

【第N轮困境/转机】（按需递进）
……

【感悟升华】
结尾方向：[与变现挂钩的方式]
```

**成功案例型骨架：**
```
【客户初始状态】
背景：[客户是谁，初始处境]
痛点：[找你之前的具体困境]

【你的价值出现】
通关密码：[你为什么是这个客户的唯一解法]
具体动作：[你做了什么]

【递进困境/转机】（按需）
……

【成功结果】
数字：[具体成果数字]
挂钩：[如何引向变现]
```

**平凡英雄型骨架：**
```
【还原小事】
场景：[具体是什么事，几句话还原现场]
特写要素：[对话/动作/数字中选]

【处理方式】
你怎么做的：[具体行动]

【人品格局】
体现了什么：[讲究/仗义/有面儿/格局大]
结尾落脚：[给观众留下什么感受]
```

输出骨架后停下来 →
「骨架确认吗？确认后逐段生成正文。有需要调整的地方直接说。」

用户确认 → 进 Step 4  
用户修改 → 调整后再次确认，不跳过

### B-Step 4：逐段生成正文

**每段生成规则（必须全部满足）：**
- 困境段：用外部冲突事件，不用内部感受（累/苦/难）
- 转机段：用具体事件，不用抽象努力（坚持/奋斗/拼搏）
- 关键时刻插入特写镜头（对话+动作+心理+数字，至少2项）
- 句子短，节奏快，口播友好
- 老郭人设规则全程应用

### B-Step 5：输出最终稿

```

总字数：[XXXX字]

[纯净版正文]
（完整正文逐字稿，无任何标注，直接可用于口播）

[导演版正文]
情绪：（激动）（压抑）（平静）（停顿）（压低声音）
镜头：[正对镜头] [特写：手部/表情] [走动] [切近景]
重读：**加粗文字 = 重读**
节奏：/ = 短停（0.5秒），// = 长停（1秒以上）
特写镜头：[特写-对话] [特写-动作] [特写-心理]（标注位置）
```

- 读取知识库中的「结构类型判断规则」，自主判断结构，不问用户
- 正文第一句必须和 hookskill 输出的最后一句自然衔接，不重复开头已说过的内容

---

### C-Step 1：自主判断结构类型

按知识库规则从三种结构中选一个：

- **四段式**：功能型/痛点解决类产品

### C-Step 2：输出骨架确认

## [D] 说观点工作流（内容类型 = 说观点）

### 核心规则

- 结构选择（归纳法/反证法）由 Claude 自主判断
- 结尾金句根据情绪基调选择模板
- 论据必须有画面感或具体数字，不允许空泛表达

---

### D-Step 1：自主分析输入

从 BOOMskill 输出中判断：
- 选题属于哪个维度：过错 / 纠结 / 委屈 / 麻烦 / 伤害
- 写作对象是谁（年轻女性/全职妈妈/结婚的女人/年轻男性/其他）
- 大多数人站在哪一方（决定观点方向）

从 hookskill 开头中判断：
- 已定基调（情绪强度、语气方向）
- 正文需要衔接的情绪节奏

### D-Step 2：自主选择结构，输出骨架确认

自主判断用归纳法还是反证法：
- **归纳法**：适合正向支持类（帮粉丝找理由/找借口）
- **反证法**：适合有争议话题（先打反例，再拉正例，解气感更强）

输出骨架：

```
【结构骨架确认】
结构类型：[归纳法/反证法]
写作对象：[XXX]
选题维度：[过错/纠结/委屈/麻烦/伤害]
观点方向：[一句话核心观点]

[归纳法骨架]
开篇观点：XXX
论据1：XXX
论据2：XXX
论据3：XXX（如有）
强化观点：XXX

[反证法骨架]
开篇观点：XXX
反例：XXX（不这样做的坏处）
正例：XXX（这样做的好处）
强化观点：XXX

请确认骨架，确认后逐段展开。
```

### D-Step 3：逐段生成，每段等确认

顺序：开篇观点 → 论据1/反例 → 论据2/正例 → 论据3（如有）→ 结尾金句

**论据质量标准（每个论据至少满足其中一条）：**
1. 有画面感的具体小故事或场景
2. 有具体数字支撑
3. 有对比反差（A这样 vs B那样）
4. 是目标粉丝的亲身经历或共同委屈

**结尾金句选择逻辑：**
- 爽快解气类 → 反问式（"不敢再说XXX了，行不行？"）
- 情感共鸣类 → 祝福式（"祝我们XXX早日XXX"）
- 立场宣言类 → 宣言式或总结式
- 警醒提示类 → 警示式

### D-Step 4：输出最终稿

```
【zhengwen-router 输出】
内容类型：说观点
结构类型：[归纳法/反证法]
总字数：[XXXX字]

[纯净版正文]
（去掉所有导演标注，只保留逐字稿正文）

---导演版分隔线---

[导演版正文]
（每句话后附加情绪/语气/停顿标注，如：[停顿0.5s] [语气加重] [语速加快] [微笑] 等）
```

## 老郭人设规则（全程应用，4种类型均适用）

**必须出现：**
- 具体数字至少1个（金额/时间/数量/比例）
- 反常识陈述至少1条（和观众预期相反的判断）
- 短句优先，口播念出来不别扭

**说话风格：**
- 直接陈述结论，不铺垫，不问候
- 用「你」不用「大家」
- 用具体场景代替抽象概念
- 金句放段落末尾，不放开头

**绝对禁止：**
- 自问自答式开头（「你有没有想过…？其实…」）
- 模糊主语（「很多人」「大部分人」「有些朋友」）
- 书面语（「然而」「因此」「综上所述」）
