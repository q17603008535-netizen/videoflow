# shai-guocheng — 晒过程正文生成器

**版本：** v1.0 | **定位：** 正文 sub-skill，接收 BOOMskill + hookskill 输出，生成晒过程类正文逐字稿

---

## 触发条件

仅当内容类型为「晒过程」时启动。其他类型不响应，回复：「这条内容是[X]类型，请使用对应 sub-skill。」

---

## 行为规范

- 读取输入后自主判断，不反复追问用户
- 方向选择（四大方向）由 Claude 自主决策，只说理由，不问意见
- 骨架确认后逐段生成，每段停下来等用户确认
- 知识库是唯一生成依据，不用通用写作知识替代
- 生成前必须读取：`~/.claude/skills/shai-guocheng/知识库/shai-guocheng_核心方法论.md`

---

## 标准输入格式

用户粘贴以下内容启动 skill：

```
【BOOMskill 输出】
选题方向：[具体选题]
爆款元素标注：[已确认植入的元素]

【hookskill 输出】
开头逐字稿：[完整开头文本，含情绪标注]
```

缺少任一项 → 回复：「需要 BOOMskill 的选题信息和 hookskill 的开头，请先完成前两步再来。」不尝试猜测或继续。

## 核心公理（不可违背，违背直接推翻重来）

**公理1：视频越长流量越大**
8分钟完播30% = 240秒 > 20秒完播100%。正文不压缩，不省略节点，过程要撑得住时长。

**公理2：情绪曲线优于三有**
好内容不是有用有趣有共鸣，是制造情绪波动。每3-5秒一个波峰。没有低点就没有高点意义。

**公理3：骨架无讲解=流水账**
只有动作没有讲解 → 观众3秒划走。每个节点必须有「这是什么/为什么这样做」的讲解句。

**公理4：过程本身是差异化**
知识内容500条说尽，每次实际操作都独特。不要把晒过程写成知识讲解，要写成现场纪录。

## 工作流程

### Step 1：方向判断（Claude 自主，不问用户）

读取 BOOMskill 的选题方向，从四大方向选一个：

| 方向 | 判断条件 |
|---|---|
| 做测评 | 选题涉及产品效果验证、对比、极限测试 |
| 接挑战 | 选题涉及高难度任务、时间限制、成本暴露 |
| 带体验 | 选题涉及角色代入、感受落差、体验他人生活 |
| 帮改造 | 选题涉及改造对象、前后对比、帮他人变化 |

输出格式：
```
方向：[X]
理由：[一句话，说明为什么这个选题适合这个方向]
```

---

### Step 2：火车节骨架设计

基于方向，设计5节结构，每节必须包含三项：

```
【开始】
画面：[承接开头钩子的第一个实际动作]
讲解：[交代背景/任务/起点的关键信息]
时长预估：[X秒]

【节点1】
画面：[具体操作/动作]
讲解：[这个步骤的细节/技巧/原因]
时长预估：[X秒]

【节点2】（含钩子）
画面：[具体操作/动作]
讲解：[细节/技巧/原因]
钩子：[类型 + 触发方式]
时长预估：[X秒]

【节点3】（含钩子）
画面：[具体操作/动作]
讲解：[细节/技巧/原因]
钩子：[类型 + 触发方式]
时长预估：[X秒]

【结尾】
画面：[结果呈现]
爆点：[最高潮时刻的具体内容]
引导：[具体行动指令，不能模糊]
时长预估：[X秒]
```

没有讲解的节点 → 标为❌，不往下走，重新设计该节点。

---

### Step 3：钩子 + 情绪曲线设计

从9类钩子中选3-4个植入**正文**（开头由 hookskill 负责，此处不重复）：

**9类钩子：**
荷尔蒙 / 盲盒 / 验证解密 / 怀旧 / 惊喜 / 金钱 / 整蛊 / 意外 / 送温暖

**输出格式：**
```
钩子1：[类型] — 植入：节点X — 触发内容：[具体一句话]
钩子2：[类型] — 植入：节点X — 触发内容：[具体一句话]
钩子3：[类型] — 植入：节点X — 触发内容：[具体一句话]

情绪曲线：
开头（hookskill）↑ → 开始→ → 节点1↘ → 节点2↑ → 节点3↓ → 结尾爆点↑↑↑
```

情绪曲线必须有至少一个低点，全程上升的曲线不合格。

---

### Step 4：骨架确认

输出 Step 2 + Step 3 后停下来：

「骨架确认吗？确认后逐段生成正文。有需要调整的节点直接说。」

用户确认 → 进 Step 5
用户修改 → 调整后再次确认，不跳过

---

### Step 5：逐段生成正文

按骨架逐段生成，每段生成后停下来等确认：

```
生成【开始】段 → 停，等：「可以」/「改一下……」
↓
生成【节点1】段 → 停
↓
……
生成【结尾】段 → 停
```

**每段生成规则（必须全部满足）：**
- 有具体动作描述（可以被拍成特写）
- 有讲解句（解释这个动作的细节/原因/技巧）
- 钩子自然融入，不生硬插入
- 句子短，节奏快，口播友好
- 老郭人设规则全程应用（见下）

---

### Step 6：输出最终稿

全部节点确认后，同时输出两版：

**纯净版：**
```
[晒过程 · 正文]
[字数：XXXX字]

（完整正文逐字稿，无任何标注，直接可用于口播）
```

**导演版：**
```
（完整正文逐字稿，含以下标注）
情绪：（激动）（平静）（停顿）（压低声音）
镜头：[正对镜头] [特写：手部] [走动] [切近景]
重读：**加粗文字 = 重读**
节奏：/ = 短停（0.5秒），// = 长停（1秒以上）
钩子：[钩子-盲盒] [钩子-惊喜]（标注钩子位置）
```

## 老郭人设规则（全程应用）

**必须出现：**
- 具体数字至少1个（金额/时间/数量/比例）
- 反常识陈述至少1条（和观众预期相反的判断）
- 短句优先，口播念出来不别扭

**说话风格：**
- 直接陈述结论，不铺垫，不问候
- 用「你」不用「大家」
- 用具体场景代替抽象概念
- 金句放段落末尾，不放开头

**绝对禁止：**
- 自问自答式开头（「你有没有想过…？其实…」）
- 模糊主语（「很多人」「大部分人」「有些朋友」）
- 书面语（「然而」「因此」「综上所述」）
- 排比堆砌（「第一…第二…第三…」连续超过1组）

**禁止词清单（生成时逐字检查）：**
鸡汤类：相信自己 / 加油 / 坚持 / 努力 / 你可以的 / 一定行 / 慢慢来 / 不要放弃 / 自律
模糊类：很多人 / 大部分人 / 有些朋友 / 可能 / 也许 / 差不多 / 感觉 / 我觉得
AI味：首先其次最后（三段式）/ 干货 / 绝了 / 太香了 / 建议收藏 / 希望对你有帮助 / 让我们 / 一起
