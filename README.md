# 短视频内容生产流水线 · Claude Skill System

基于 Claude Code Skill 机制构建的短视频内容生产系统。从 IP 人设建档到逐字稿输出，全流程由方法论驱动，AI 执行。

---

## 整体架构

```
video-flow（总入口，自动检测阶段）
│
├── ip-archive          建档阶段：录音 → IP人设档案
│
├── boom-elements       选题阶段：原始选题 → 爆款选题
│
├── hook-anchor         开头阶段：选题+人群 → 开头钩子
│
└── zhengwen-router     正文阶段：选题+开头 → 完整逐字稿
        │
        ├── shai-guocheng   晒过程正文
        ├── jiang-gushi     讲故事正文
        ├── shuo-chanpin    说产品正文
        └── shuo-guandian   说观点正文
```

---

## 使用流程

### 方式一：使用总入口（推荐）

触发 `/video-flow`，粘贴内容或描述需求，系统自动检测阶段并执行对应流程。

### 方式二：直接触发对应阶段

```
新号起步        →  /ip-archive
优化选题        →  /boom-elements
生成开头        →  /hook-anchor
生成正文        →  /zhengwen-router
```

---

## 数据流

```
录音稿
  ↓ ip-archive
【IP人设档案】+ BOOMskill直通参数（内容形式 + 目标人群）
  ↓ 用户带入
boom-elements
  ↓
【BOOMskill 输出】（选题方向 + 爆款元素标注）+ 人群属性摘要
  ↓ 用户带入
hook-anchor
  ↓
【hookskill 输出】（开头逐字稿）
  ↓ 用户拼接两个输出块
zhengwen-router
  ↓ 自动路由到对应子类型
逐字稿（纯净版 + 导演版）
```

## Skill 说明

### video-flow — 流水线总入口
检测用户所处阶段（建档/选题/开头/正文），告知用户后自动执行对应流程。支持直接粘贴任意阶段的输出内容触发。

### ip-archive — IP人设档案构建
读取录音转文字稿，提取人设八维信息，将素材归类到四大内容方向（讲故事/晒过程/说产品/说观点），诊断主攻方向，输出结构化人设档案。支持持续更新。

### boom-elements — 选题爆款元素改造
接收原始选题，套用八大爆款元素框架（成本暴露/人群标签/荷尔蒙/怀旧/头牌效应/奇葩反差/反向操作/难吃难用差），输出改造版高流量选题。

### hook-anchor — 开头钩子生成
基于人群属性推导人性弱点，提炼情绪痛点，生成符合目标人群的开头钩子。只做开头，不做正文。

### zhengwen-router — 正文生成路由器
接收选题和开头，自动判断内容类型，路由到对应正文工作流，输出完整逐字稿（纯净版 + 导演版）。内置文案写作规则约束输出质量。

### shai-guocheng — 晒过程正文生成
适用于：测评/挑战/体验/改造类内容。核心方法：火车节骨架 + 细节讲解 + 钩子埋设 + 情绪曲线设计。

### jiang-gushi — 讲故事正文生成
适用于：个人成就/成功案例/平凡英雄类内容。核心方法：困境转机结构 + 特写镜头 + 变现挂钩。

### shuo-chanpin — 说产品正文生成
适用于：功能型/低价型/颜值型产品内容。核心方法：四段式/价格对比/视觉展示三种结构按产品类型匹配。

### shuo-guandian — 说观点正文生成
适用于：表达立场/支持某类人/输出判断类内容。核心方法：归纳法/反证法结构 + 论据质量标准 + 结尾金句逻辑。

## 内容类型对照

| 内容类型 | 核心逻辑 | 适用场景 |
|---|---|---|
| 晒过程 | 火车节 + 钩子 + 情绪曲线 | 手艺/服务/产品制作过程展示 |
| 讲故事 | 困境转机 + 特写镜头 | 个人经历/客户案例/小事格局 |

| 说观点 | 支持粉丝 + 论据支撑 + 金句 | 表态/立场/情绪共鸣 |

## 依赖说明

各正文 sub-skill 的知识库已内置于对应 `知识库/` 目录。

zhengwen-router 在执行所有正文工作流时，额外读取：
```
zhengwen-router/知识库/wenxie_写作规则.md
```
该文件提炼自文案写作方法论，约束逐字稿输出质量（特写镜头、客观事实、动词名词量词优先）。

---

## 设计原则

- **方法论驱动**：每个 skill 的生成依据来自知识库，不依赖 AI 通用能力
- **接口标准化**：所有 skill 的输入输出格式统一，支持跨 skill 数据传递
- **职责单一**：每个 skill 只做一件事，不越界
- **自主判断优先**：AI 自主读内容做判断，不逼问用户提供信息
