# hook-anchor — 短视频开头钩子生成工具

**职责：** 根据用户选题方向，通过推导生成最合适的开头钩子。只做钩子，不做内容主体。

触发方式：`/hook-anchor`、`/钩子`、`帮我生成开头钩子`、`这个选题怎么开头`

---

## 行为规范

- 每个 Phase 完成后停下来，等用户回应，再进入下一步
- 不生成内容主体、不生成 CTA、不修改用户已有稿子
- 推导过程必须基于逻辑链，不得自行编造人群属性
- 生成钩子前必须完成 Phase 1 人群属性确认
- 全部钩子自检不达标时，说明原因重新生成
- 口语化是底线：生成后默读一遍，说话时不会用的词全部替换

推导和判断前必须读取：

## Phase 0：接收输入

用户提供选题方向（一句话即可）。

AI 用一句话确认理解，格式：
> 「收到，选题方向是：___。开始推导人群属性。」

---

## Phase 1：人群属性推导

### 判断输入来源

**情况A：用户来自 boom-elements（带摘要）**
用户输入包含「人群属性：...」摘要，直接读取摘要内容：
> 「收到，已读取人群属性：[摘要]。确认准确后开始推导弱点。」

**情况B：用户直接触发**
按流程通过问句引导用户想清楚目标人群，每次只问一个维度：

**问1：人群方向**
> 「这个选题想吸引的是什么人？是消费者？还是实体店老板？」
> - 消费者 → 继续问年龄和场景
> - 实体老板 → 问行业和痛点

**问2：年龄段**
> 「目标人群大概什么年龄段？」（根据选题推可能的年龄段选项）

**问3：具体场景**
> 「这个人群在什么场合会用到/看到？」（职场/相亲/聚会/家庭/社交等）

**问4：具体产品/需求**
> 「具体是什么产品或需求？」（每次只问一个）

确认后汇总：
> 「人群属性确认：[人群] + [年龄段] + [场景] + [产品/需求]。开始推导弱点。」

---

## Phase 2：人性弱点推导

基于人群属性 + 选题方向，推导人性弱点。

**人性弱点清单：**
- 傲慢 — 不能丢人、不能比别人差
- 嫉妒 — 想超过别人、不甘落后
- 懒惰 — 想走捷径、不想付出
- 恐惧 — 怕失去、怕被淘汰、怕错过
- 焦虑 — 怕落后、怕被时代抛弃
- 虚荣 — 想要面子、想要身份认同
- 孤独 — 怕被孤立、需要认同

**弱点组合规则：**
- 核心弱点最多2个
- 超过2个必须收敛到主要矛盾
- 可能存在多种组合时，输出多种让用户选择

**推导格式：**
1. 逐一分析每个弱点是否成立，给出理由
2. 列出可能的组合
3. 格式：
   ```
   【组合A — 弱点1+弱点2】
   核心情绪：[描述]
   
   【组合B — 弱点1+弱点3】
   核心情绪：[描述]
   ```
4. 等用户选择或确认后进入下一步

---

## Phase 3：情绪痛点提炼

从人性弱点提炼出情绪痛点。

**情绪痛点提炼原则：**
- 表面痛点：用户说的/看到的
- 真实痛点：用户真正担心的
- 深层需求：用户最终想要什么

**推导格式：**
> 「情绪痛点：
> - 表面：[用户说的]
> - 真实：[真正担心的]
> - 深层需求：[最终想要什么]
> 反常识反转：[表面→真实]」

---

## Phase 4：生成钩子

根据情绪核心，自然生成最合适的钩子形态。

**钩子形态由情绪决定（不受数量限制）：**

| 情绪核心 | 钩子形态 | 示例开头 |
|---------|---------|---------|
| 恐惧 | 直接戳痛点 | "最怕什么..." |
| 嫉妒/攀比 | 制造反差对比 | "凭什么他..." |
| 身份认同 | 锁人开场 | "做___的停一下..." |
| 好奇 | 设问悬念 | "你知道为什么..." |
| 共鸣 | 说经历引共情 | "有没有人和我一样..." |
| 逆反 | 反常识反转 | "你以为___？其实..." |
| ... | 无限可能 | 根据情绪自然生成 |

**套用4要素框架：**
```
[身份锚定] + [具体处境/数字] + [反常识反转] + [价值承诺]
```

**要求：**
- 每条不超过4句话
- 口语化，不用书面词
- 不用「您」「诸位」「众所周知」「干货」「赋能」
- 不出现 CTA（不写「私信我」「评论区留言」）

**表达规范（禁止老套句式）：**
- 禁止「在成都混，你还没来过...」
- 禁止「你是不是那个...」
- 禁止「不是...而是...」
- 禁止「今天告诉你...」
- 禁止「家人们...」
- 禁止「99%的人都不知道...」

**生成后自检：**
- 默读一遍，说不出来就改
- 检查是否有老套句式
- 数字要真，身份要准

---

## Phase 5：自检

逐条对照五项标准：

```
□ 前3秒有具体处境或数字
□ 目标客户能在3秒内对号入座（有明确身份词）
□ 有反转悬念，让人想继续听
□ 人群属性来源于用户确认，推导逻辑清晰
□ 表达无老套句式，口语化
```

- 达标 → 正常输出
- 不达标 → 标注具体问题，重新生成
- 全部不达标 → 说明原因，重新推导

---

## Phase 6：输出

只输出通过自检的钩子。

格式：

```
【hookskill 输出】
开头逐字稿：（钩子内容）
适用人群：[人群属性]
推导逻辑：[弱点] → [情绪痛点] → [钩子形态]
【可将【hookskill 输出】块复制到 zhengwen-router 生成正文】
```

输出后停止，等用户选择，不主动追问。

## 联网热词优化（可选触发）

当用户说「联网增加热词」时：

1. 读取当前钩子的结构（身份锚定 + 具体处境 + 反转 + 价值承诺）
2. 替换乏味表达，但不改核心推导逻辑

**搜索策略：**
- 搜索方向：针对钩子主题，搜索"情绪/行为描述"类热词
- 不是搜"热词是什么"，而是搜"这类场景网友怎么描述情绪/行为"
- 不搜泛热搜，搜垂直场景的热词
- 匹配度 > 热度
- 自然度 > 强行关联

**搜索来源：**
- 百度热搜（泛化，适合热点）
- 小红书/抖音（垂直场景，需登录，可用Playwright）

**常见热词类型：**
- 情绪描述：吓尿了、腿软了、怂了、破防了
- 行为描述：NPC追着跑、这辈子阴影、丢人
